用户表 (users)
存储系统用户的基本信息，支持角色权限管理。仅管理员可注册新用户。
字段名 数据类型 约束 描述
name VARCHAR(100) 唯一 非空 分类名称
description TEXT 可空 分类描述
parent_id INTEGER 外键(自关联) 父分类ID，支持层级分类
created_at TIMESTAMP 默认当前时间 创建时间
📦
材料表 (materials)
管理实验室材料库存，包括规格参数、供应商信息和库存跟踪。
字段名 数据类型 约束 描述
id INTEGER 主键 材料唯一标识
name VARCHAR(100) 非空 材料名称
category_id INTEGER 外键 材料分类ID
brand VARCHAR(100) 可空 品牌
model VARCHAR(100) 可空 型号规格
specifications JSON 可空 技术参数规格
unit VARCHAR(20) 非空 计量单位 (个/瓶/克/升等)
total_quantity DECIMAL(10,3) 默认0 总库存数量
available_quantity DECIMAL(10,3) 默认0 可用库存数量
reserved_quantity DECIMAL(10,3) 默认0 预留库存数量
min_stock_alert DECIMAL(10,3) 可空 最低库存预警阈值
location VARCHAR(200) 可空 存放位置
supplier VARCHAR(100) 可空 供应商
purchase_price DECIMAL(10,2) 可空 单价
expiry_date DATE 可空 过期日期
safety_info TEXT 可空 安全注意事项
owner_id INTEGER 外键 负责人ID
last_used_at TIMESTAMP 可空 最后使用时间
created_at TIMESTAMP 默认当前时间 创建时间
updated_at TIMESTAMP 可空 更新时间

id INTEGER 主键 分类唯一标识
name VARCHAR(100) 唯一 非空 分类名称

使用日志表 (usage_logs)
记录设备和材料的使用情况，支持半自动化日志记录。
字段名 数据类型 约束 描述
id INTEGER 主键 日志唯一标识
user_id INTEGER 外键 操作用户ID
resource_type ENUM 非空 资源类型: equipment/material
resource_id INTEGER 非空 资源ID (设备ID或材料ID)
action ENUM 非空 操作类型: start_use/end_use/consume/maintenance
quantity_used DECIMAL(10,3) 可空 使用数量 (材料专用)
duration_minutes INTEGER 可空 使用时长(分钟，设备专用)
purpose VARCHAR(200) 可空 使用目的
notes TEXT 可空 备注信息
issues_reported TEXT 可空 问题反馈
project_name VARCHAR(100) 可空 关联项目名称
timestamp TIMESTAMP 默认当前时间 操作时间
auto_recorded BOOLEAN 默认false 是否自动记录

设备和材料的预约管理，支持时间冲突检测和审批流程。
字段名 数据类型 约束 描述

resource_id INTEGER 非空 资源ID (设备ID或材料ID)
start_time TIMESTAMP 非空 预约开始时间

quantity_reserved DECIMAL(10,3) 可空 预约数量 (材料专用)

🔗 数据表关系图谱

users ↔ roles
users.role_id → roles.id (多对一)
一个用户对应一个角色，一个角色可以被多个用户使用
users ↔ user_sessions
user_sessions.user_id → users.id (多对一)
一个用户可以有多个会话记录
equipment ↔ equipment_categories
equipment.category_id → equipment_categories.id (多对一)
一个设备属于一个分类，一个分类包含多个设备
equipment ↔ users (负责人)
equipment.responsible_user_id → users.id (多对一)
一个设备有一个负责人，一个用户可以负责多个设备
materials ↔ material_categories
materials.category_id → material_categories.id (多对一)
一个材料属于一个分类，一个分类包含多个材料
materials ↔ users (负责人)
materials.owner_id → users.id (多对一)
一个材料有一个负责人，一个用户可以负责多个材料
usage_logs ↔ users
usage_logs.user_id → users.id (多对一)
一条日志对应一个用户，一个用户可以有多条日志
usage_logs ↔ equipment/materials
usage_logs.resource_id → equipment.id 或 materials.id (根据resource_type)
使用多态关联，通过resource_type区分资源类型
reservations ↔ users (预约者)
reservations.user_id → users.id (多对一)
一个预约对应一个预约用户
reservations ↔ users (审批者)
reservations.approved_by → users.id (多对一)
一个预约对应一个审批用户
reservations ↔ equipment/materials
reservations.resource_id → equipment.id 或 materials.id (根据resource_type)
预约使用多态关联，通过resource_type区分资源类型
分类表自引用关系
equipment_categories.parent_id → equipment_categories.id
material_categories.parent_id → material_categories.id
支持层级分类结构

SQL 建表语句预览
基于 PostgreSQL 语法的建表语句，可根据实际需求调整为 SQLite 语法。
-- 角色表 CREATE TABLE roles ( id SERIAL PRIMARY KEY, name VARCHAR(50) UNIQUE NOT NULL, description TEXT, permissions JSONB, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- 用户表 CREATE TABLE users ( id SERIAL PRIMARY KEY, username VARCHAR(50) UNIQUE NOT NULL, email VARCHAR(100) UNIQUE NOT NULL, hashed_password VARCHAR(255) NOT NULL, full_name VARCHAR(100), phone VARCHAR(20), role_id INTEGER REFERENCES roles(id), is_active BOOLEAN DEFAULT TRUE, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, last_login_at TIMESTAMP ); -- 用户会话表 CREATE TABLE user_sessions ( id SERIAL PRIMARY KEY, user_id INTEGER REFERENCES users(id) ON DELETE CASCADE, token_hash VARCHAR(255) UNIQUE NOT NULL, expires_at TIMESTAMP NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, ip_address INET, user_agent TEXT ); -- 设备分类表 CREATE TABLE equipment_categories ( id SERIAL PRIMARY KEY, name VARCHAR(100) UNIQUE NOT NULL, description TEXT, parent_id INTEGER REFERENCES equipment_categories(id), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- 设备表 CREATE TABLE equipment ( id SERIAL PRIMARY KEY, name VARCHAR(100) NOT NULL, model VARCHAR(100), serial_number VARCHAR(100) UNIQUE, category_id INTEGER REFERENCES equipment_categories(id), status VARCHAR(20) DEFAULT 'available' CHECK (status IN ('available', 'in_use', 'maintenance', 'broken')), location VARCHAR(200), manual_url TEXT, specifications JSON, purchase_date DATE, purchase_price DECIMAL(10,2), responsible_user_id INTEGER REFERENCES users(id), last_used_at TIMESTAMP, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP ); -- 维护记录表 CREATE TABLE maintenance_records ( id SERIAL PRIMARY KEY, equipment_id INTEGER REFERENCES equipment(id) ON DELETE CASCADE, maintainer_id INTEGER REFERENCES users(id), maintenance_type VARCHAR(20) NOT NULL, description TEXT, maintenance_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP, next_maintenance_date TIMESTAMP );

设备表 (equipment)
管理实验室所有设备信息，包括状态跟踪、使用说明和维护记录。
字段名 数据类型 约束 描述
id INTEGER 主键 设备唯一标识
name VARCHAR(100) 非空 设备名称
model VARCHAR(100) 可空 设备型号
serial_number VARCHAR(100) 唯一 设备序列号
category_id INTEGER 外键 设备分类ID
status ENUM 默认'available' 状态: available/in_use/maintenance/broken
location VARCHAR(200) 可空 设备位置
manual_url TEXT 可空 使用说明文档链接
specifications JSON 可空 技术规格参数
purchase_date DATE 可空 购买日期
purchase_price DECIMAL(10,2) 可空 购买价格
responsible_user_id INTEGER 外键 负责人ID
last_used_at TIMESTAMP 可空 最后使用时间
created_at TIMESTAMP 默认当前时间 创建时间
updated_at TIMESTAMP 可空 更新时间

维护记录表 (maintenance_records)
记录设备的维护、检修和保养历史。
字段名 数据类型 约束 描述
id INTEGER 主键 维护记录唯一标识
